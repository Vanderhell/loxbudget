# V1.1 Footprint Notes (Causality)

This document tracks footprint impact when compiling with `LOXBUDGET_ENABLE_CAUSALITY=1`.

## Storage impact

When enabled, `LOXBUDGET_REQUIRED_SIZE()` adds:

- `4B` header
- `LOXBUDGET_CAUSALITY_MAX_EDGES * 4B` edge array
- `(max_ops + 7) / 8` bytes visited bitmap

Defaults:

- `LOXBUDGET_CAUSALITY_MAX_EDGES = 32` → `128B`
- `LOXBUDGET_CAUSALITY_MAX_DEPTH = 3` (no direct storage cost)

So default extra storage is roughly:

`~132B + (max_ops+7)/8`

## Code size impact

Code size depends on toolchain and LTO, but should remain opt-in:

- `LOXBUDGET_ENABLE_CAUSALITY=0` → causality code should be removable by the linker (no overhead).
- `LOXBUDGET_ENABLE_CAUSALITY=1` → adds graph + DFS walk code.

## How to measure (ARM cross build)

If you have `arm-none-eabi-*` toolchain installed:

- Build the object with causality enabled (example):
  - `make cross-full LOXBUDGET_ENABLE_CAUSALITY=1`
- Compare `size` output against causality disabled.

Record numbers here once collected for a real target (MCU, compiler, flags).
