# loxbudget v1.0.0

`loxbudget` `v1.0.0` is now released.

This release marks the first stable public version of a tiny no-heap C99 admission-control library for embedded firmware. It is designed for pre-flight decisions before risky work such as MQTT publish bursts, OTA updates, flash writes, logging spikes, or other operations that should be allowed, degraded, delayed, or rejected under pressure.

## Highlights

- Stable public API for `check`, `enter`, `leave`, resource setup, and operation policy control
- No heap, no floats, no global mutable state
- Optional calibration workflow and host-side reporting
- Optional adapters for microlog, microhealth, microconf, microbus, nvlog, and loxguard
- Single-header distribution and CMake package support
- CI coverage across GCC, Clang, Cortex-M0 cross-builds, sanitizers, fuzz smoke, coverage, and footprint checks

## What this release is for

`v1.0.0` is the baseline stable release for integration into embedded firmware projects that need deterministic admission control without pulling in a scheduler, allocator, watchdog, or RTOS replacement.

## What comes next

Post-`v1.0.0` work is expected to focus on stronger hardware evidence, longer validation runs, and deciding the exact scope of a future `v1.1.0` cycle.
