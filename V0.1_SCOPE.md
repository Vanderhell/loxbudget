# V0.1 — Frozen Scope

> **This document is FROZEN.** Any change to the V0.1 scope requires explicit consensus and a new freeze date below.
> **Frozen on**: project inception
> **Status**: scope locked. Implementation may begin.

---

## Purpose

V0.1 exists to prove **one thing**:

> The core decision engine is small, deterministic, and trustworthy.

Nothing else. Not feature richness. Not adapter integration. Not a polished example gallery. Just: small, deterministic, trustworthy core.

If V0.1 ships and that claim holds, the project has a foundation. If not, no amount of features in V0.2+ will save it.

## In Scope (V0.1)

loxbudget_status_t loxbudget_set_resource(loxbudget_t*,

loxbudget_status_t loxbudget_op_set_need(loxbudget_t*,

### Decision engine

- All five action types: `ALLOW_FULL`, `ALLOW_DEGRADED`, `WAIT`, `REJECT`, `LOCKDOWN`.
- All five pressure states: `NORMAL`, `ELEVATED`, `CRITICAL`, `SURVIVAL`, `LOCKDOWN`.
- Per-operation pressure-to-action mapping.
- All three resource kinds: `REUSABLE`, `CONSUMABLE`, `STATE`.
- Atomic multi-resource reservation in `enter`.
- Lease validation via per-instance magic.
- Watermark tracking on resources.
- Decision counters in snapshot.

### HAL

- Weak default symbols for: `now_ms`, `critical_enter`, `critical_exit`, `boot_proven`, `voltage_ok`, `network_up`.
- Callback HAL via `cfg->hal_callbacks` (overrides weak symbols).
- Fail-closed mode (`hal_strict = 1`) is default.
- `loxbudget_hal_default_permissive()` accessor for tests/minimal example.

### Storage

- User-supplied buffer.
- `LOXBUDGET_REQUIRED_SIZE(n_res, n_ops, audit)` macro returning compile-time constant.
- Alignment check in `init`.
- Magic value validation for re-init detection.
- Per-instance lease magic.

### Source layout

- **One** core C file: `src/loxbudget.c` (~600 LOC target).
- One HAL file: `src/loxbudget_hal.c` (~50 LOC).
- One header: `include/loxbudget.h`.
- Splitting into multiple files is **forbidden** in V0.1.

### Tests (8 mandatory cases)

1. `test_init_invalid_args` — NULL, zero size, misalignment, bad config.
2. `test_init_valid` — succeeds with proper inputs, snapshot reflects empty state.
3. `test_decision_is_pure` — same input → same output (determinism invariant).
4. `test_no_partial_reservation` — failed `enter` leaves all resources untouched.
5. `test_lease_lifecycle` — enter/leave normal flow, double-leave detected.
6. `test_lease_magic_per_instance` — lease from instance A rejected by instance B.
7. `test_unknown_op_graceful` — unknown op_id returns REJECT/UNKNOWN_OP, no UB.
8. `test_hal_strict_fail_closed` — `register_op` with `OPF_REQUIRES_BOOT_PROVEN` and missing HAL → `LOXBUDGET_ERR_HAL_NOT_CONFIGURED`.

Plus: compile-time `_Static_assert` checks on all public struct sizes.

### Example

- Exactly one: `examples/01_bare_metal_minimal/main.c`.
- ≤ 50 lines including includes, comments, and `main`.
- Demonstrates init, set_resource, register_op, op_set_need, check, response.
- Uses `loxbudget_hal_default_permissive()`.

### Build

- One Makefile that builds host tests with GCC and Clang.
- One `CMakeLists.txt` for users who want CMake integration.
- Compile-only verification for `arm-none-eabi-gcc` (Cortex-M0).
- No amalgamation script (the library is already one file).

### CI gates

1. GCC host build + tests pass.
2. Clang host build + tests pass.
3. MSVC host build + tests pass (with callback HAL).
4. `arm-none-eabi-gcc` cross-compile succeeds for Cortex-M0.
5. `tools/check_banned_symbols.sh` passes on all binaries.
6. Library `.bss = 0` enforced.
7. Footprint < 4 KiB on Cortex-M0+ -Os (TINY profile).
8. Minimal example ≤ 50 lines (line count check in CI).
9. `clang-format` clean.
10. `clang-tidy` clean (no new warnings).
11. UBSan/ASan clean on host tests.

## Risks Specific to V0.1

| Risk                                              | Mitigation                                  |
|---------------------------------------------------|---------------------------------------------|
| Atomic enter has race conditions in ISR context   | Document as not-thread-safe by default; HAL critical section is opt-in. |
| Footprint exceeds 4 KiB on Cortex-M0              | Start measuring early; aggressive use of `-ffunction-sections` and LTO. |
| `_Static_assert` not portable to all toolchains   | Document required toolchain features; provide fallback macro. |
| Weak symbols don't behave on MSVC                 | Callback HAL is the MSVC path; CI verifies. |
| Scope creep into V0.2 features                    | This document. Reject PRs that don't match. |
| Tests too coupled to implementation               | Test behavior via public API only. No internal access. |
